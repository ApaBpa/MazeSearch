# Code standards
Just for me to remember

## Formatting
- UNIX-style line endings (LF)
- Full tab indent
- Newline at EOF

## Naming
### C++
- `snake_case` for vars
- `PascalCase` for classes/functions
- Never use namespaces

### HTML/JS
- `snake_case` for vars
- `camelCase` for classes/functions

## Comments
- Function header + params
- TODO marker

## Build
- `make maze_omp`
